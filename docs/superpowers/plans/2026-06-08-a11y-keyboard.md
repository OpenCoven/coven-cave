## Task 3: Glyph picker — roving (both axes) + scrollIntoView

**Why:** Audit P0 — the grid of ~800 GlyphButton renders is unusable from keyboard (sequential tab through 800 items). Roving + visible-scroll fixes both.

**Special concern:** with 800 buttons, focusing item N requires it to be visible. The hook focuses programmatically, but if the focused item is below the fold, the browser may or may not auto-scroll. We add an explicit `scrollIntoView({ block: "nearest" })` after focus.

**Files:**
- Modify: `src/components/familiar-glyph-picker-panel.tsx`
- Create: `src/components/glyph-picker-roving.test.ts`

### Step 3.1 — Inspect structure

```bash
sed -n '100,160p' src/components/familiar-glyph-picker-panel.tsx
```

Note: the grid container element, the `GlyphButton` class, the columns-per-row count (informs ArrowLeft/Right vs ArrowUp/Down delta).

### Step 3.2 — Confirm column count for vertical step

In the grid CSS (likely `globals.css` or a `.glyph-picker-grid` rule), find `grid-template-columns`. The ArrowUp/Down step needs to be the number of columns. If the grid is `repeat(auto-fill, minmax(48px, 1fr))`, columns are responsive — call this out and use a column-agnostic strategy: ArrowDown moves to the item directly below by computing `rect.top` of the next visible row's first item. Easier: drop vertical arrow support and only support ArrowLeft/Right + Home/End for a 1D rove. The grid is visually 2D but logically 1D ordering.

**Decision (lock in here, document in the test):** Use 1D rove (left/right + home/end across all 800). Vertical arrows are unsupported in this PR; if column-aware nav is desired later, ship as PR D.

## Task 5: Board table — arrow nav + Enter to open + Esc to deselect

f. Add Enter + Escape handling at the table level (not the tbody — keyboard events on rows bubble to tbody, but the hook owns arrow keys; Enter/Escape don't conflict with the hook):

```tsx
useEffect(() => {
  const tbody = tbodyRef.current;
  if (!tbody) return;
  function onKey(e: KeyboardEvent) {
    if (e.key === "Enter") {
      const rows = tbody.querySelectorAll<HTMLTableRowElement>('tr[data-board-row="true"]');
      const row = rows[activeIndex];
      const cardId = row?.dataset.cardId; // we'll set this in the row markup
      if (cardId) {
        e.preventDefault();
        onSelect(cardId);
      }
    } else if (e.key === "Escape") {
      // Deselect — clear selection. Parent owns selection; if there's an
      // onDeselect prop, use it; otherwise call onSelect(""):
      e.preventDefault();
      onSelect(""); // adapt to actual selection-clear signature
    }
  }
  tbody.addEventListener("keydown", onKey);
  return () => tbody.removeEventListener("keydown", onKey);
}, [activeIndex, onSelect]);
```

>
