# Coven Cave — full-app UX audit

**Date:** 2026-06-08
**Status:** Findings only (no edits). Punch list pending triage.
**Scope:** Every shipped surface — theme tokens, shell chrome, home/chat, agents/familiars, board/calendar/inbox, library/salem/browser/terminal, settings/aesthetic/onboarding/marketplace.
**Axes weighted equally:** accessibility · visual hierarchy & consistency · interaction & motion · non-happy-path & IA.

This audit is the punch list a world-class polish pass would work from. It is intentionally evidence-led — every P0/P1 cites a file:line so the fix has a target. It does **not** propose IA changes (the approved [shell IA spec](./2026-06-08-ui-ux-shell-ia-design.md) covers that) and it does **not** redo themes (the approved [light/dark + 8 themes spec](./2026-06-08-light-dark-mode-and-8-themes-design.md) covers that). It catches the work *between* those: keyboard, focus, motion, error states, missed token coverage, and the small consistencies that separate "shipped" from "felt".

---

## Cross-cutting themes

Ten patterns recurred across surfaces. Fix once, fix everywhere — these are higher leverage than any single P0.

1. **Modals/drawers don't trap focus.** Confirmed missing in command palette (`command-palette.tsx:120`), board inspector (`board-inspector.tsx:770`), library reader (`library-doc-preview.tsx:504`), GitHub attach/handoff (`library-github-list.tsx:202`), onboarding overlay (`onboarding-overlay.tsx:484`), glyph picker (`familiar-glyph-picker.tsx`), chat attachment lightbox (`chat-view.tsx:1268`). Build one `useFocusTrap` hook + restore-on-close, apply everywhere.
2. **No `aria-live` regions for dynamic content.** Chat transcript (`chat-view.tsx:885`), inbox-toast (`inbox-toast.tsx:42`), board optimistic errors, salem replies, board multi-select count, settings save feedback — none announce. Add a global `<LiveRegion>` and route assertive errors / polite progress through it.
3. **`prefers-reduced-motion` is spotty.** Honored in globals skeleton (`globals.css:1435`) but skipped by `ThinkingIndicator` (`chat-view.tsx:1066`), board drawer (`board.css:204`), reader slide-up, command palette fade (`command-palette.tsx:302`), salem widget, scroll-to-bottom smooth scroll (`chat-view.tsx:461,926`), mode-toggle transition, `.ui-btn-spinner` (`globals.css:1296`). Wrap all keyframes/transitions in the existing media query.
4. **No roving tabindex in grid-like surfaces.** Glyph picker grid (`familiar-glyph-picker-panel.tsx:193`), familiar studio tabs (`familiar-studio.tsx:94`), avatar rail (`familiar-avatar-rail.tsx`), board kanban columns/rows (`board-table.tsx`, `board-kanban.tsx`), eval-loop track-filter (`eval-loop-panel.tsx:189`), agents detail rail (`agents-view.tsx:479`). One `useRovingTabIndex` hook closes all of these.
5. **No keyboard alternative for drag.** Kanban is mouse-only (`board-kanban.tsx:225`). Hard P0 for keyboard users.
6. **Icon buttons missing accessible names or focus styles.** Notification gear (`notification-bell.tsx:151`), board sort caret (`board-table.tsx:123`), familiar status badges (`plugin-card.tsx:88`), browser tab without `aria-current="page"`. Audit every `<button>` that has no text node.
7. **Hardcoded color literals persist.** Salem section in `globals.css:3537–3697` leaks 15+ hex/rgba (the recent #255 sweep missed this). 26 component files still use `text-white`/`bg-white`/`text-black`/`bg-black` in Tailwind classes — top offenders: `automations-view.tsx` (~8), `chat-surface.tsx`, `calendar-view.tsx`, `agents-view.tsx`, `memory-graph-3d.tsx`, `browser-pane.tsx`, `familiar-studio-look-tab.tsx`. Modal backdrops also hardcode `rgba(0,0,0,…)` and `bg-black/40` (`library.css`, `github-view.tsx`).
8. **Empty / error states improvise per surface.** AgentsEmptyState, ChatEmptyState, board empty, library empty, inbox zero-state, capabilities empty, marketplace empty — each invents its own copy, icon, and CTA. No shared `<EmptyState>` / `<ErrorState>` primitive.
9. **3D canvases lack text alternatives.** `memory-graph-3d.tsx:428`, `trace-graph-3d.tsx:76` respect reduced-motion (✓) and trap Esc/Home (✓), but have no text-mirror, no keyboard orbit/zoom, no per-selection announcement, no fallback non-3D view.
10. **Form errors not associated with inputs.** Settings import URL (`settings-shell.tsx:713/745`), onboarding glyph (`onboarding-overlay.tsx:896`), vault add (`vault-panel.tsx:82–129`), new-card-modal (`new-card-modal.tsx:154`). Missing `aria-describedby`, no `aria-invalid`, no required indicators.

## P0 — broken / blocking

Order: assistive-tech blockers first, then keyboard-only blockers, then visible quality issues.

### Accessibility / semantics

- **Chat transcript has no `aria-live` and no role.** `chat-view.tsx:885` — screen-reader users get no announcement of streamed turns. Add `role="log"` + `aria-live="polite"` + `aria-relevant="additions"` on the thread container; ensure off-screen messages aren't re-announced on history-load.
- **Chat textareas have no programmatic label.** `chat-view.tsx:1002`, `home-composer.tsx:235` — placeholder isn't a label. Add `aria-label` or visually-hidden `<label htmlFor=…>`.
- **Command palette input unlabeled; results aren't a listbox.** `command-palette.tsx:313,324` — input has no `aria-labelledby`; results are `<ul><button>` with `aria-current="true"` (non-standard). Convert to `role="listbox"` + `role="option"` + `aria-activedescendant` on the input.
- **Glyph-picker grid is sequential tab.** `familiar-glyph-picker-panel.tsx:193–223` — ~800 visible buttons in tab order, no roving tabindex, no arrow nav. Effectively unusable from keyboard.
- **Salem search input + library search input have no labels.** `salem-widget.tsx:176`, `library-doc-list.tsx:63`, plus timeline filter selects (`library-timeline.tsx:110–140`).
- **Onboarding overlay has no focus trap and no `Escape`.** `onboarding-overlay.tsx:484` — focus escapes to background; Escape doesn't dismiss.
- **xterm.js terminal has no SR fallback.** `bottom-terminal.tsx` — canvas only. Wrap with `role="region" aria-live="polite" aria-label="Terminal output"` + an off-screen mirror of recent PTY output.
- **Board kanban is mouse-only drag.** `board-kanban.tsx:225–234` — keyboard users cannot move cards across statuses. Add keyboard-reorder (Space to grab, arrows to move, Space to drop) or at minimum a "Move to…" menu per card.
- **Board inspector / new-card / filter popover lack proper dialog/menu semantics.** `board-inspector.tsx:770`, `new-card-modal.tsx:154`, `board-filter-popover.tsx:82` — no focus trap, no `role="dialog"`/`role="menu"`, no return-focus.
- **Plugin status badge is colour-only.** `plugin-card.tsx:88–103` — "Installed/Updating/Skill" not announced. Add `aria-label`.

### Visual / token

- **Scrollbar uses raw purple rgba.** `globals.css:3673` — `scrollbar-color: rgba(124, 77, 255, 0.3) transparent` ignores theme. Tokenize.
- **Light-mode `--ring-focus` hardcodes `#6F62A8`.** `globals.css:200` — desync risk vs. `--accent-presence`. Derive.
- **Salem CSS leaks 15+ hex/rgba.** `globals.css:3537–3697` — breaks theme switching for that whole panel.
- **Top-bar Settings icon button gets `outline:none` without a focus-visible replacement on some paths.** `top-bar.tsx:75` — keyboard users lose the focus indicator. Verify `.top-bar__icon-btn:focus-visible` actually fires.

### Motion

- **`.ui-btn-spinner` ignores `prefers-reduced-motion`.** `globals.css:1296` — global rule covers `::before/::after` only, not the named keyframe.

## P1 — significant rough edges

### Chat / home

- **No "new message" pill or unread badge while scrolled away.** `chat-view.tsx:919–931` — scroll-to-bottom FAB exists but doesn't say *why* you'd press it.
- **Error banner is a plain div.** `chat-view.tsx:934` — needs `role="alert"` and a visible icon.
- **Home-composer swallows send errors silently** (only toast, which has no aria-live yet). `home-composer.tsx:143–163`.
- **Code-block copy button is opacity-0 until hover.** `cave-chat.css:349` — focus-visible reveal exists but the button is hidden from initial DOM render, so keyboard discovery fails.
- **`ThinkingIndicator` doesn't explain what's happening.** `chat-view.tsx:1066–1095` — "thinking" is jargon; surface tool-use vs. model-response distinction.
- **Truncated attachments don't say how much was cut.** `chat-view.tsx:1281–1286,1337`.

### Shell

- **Sidebar collapse toggle has no `aria-expanded`.** `familiar-avatar-rail.tsx:232`.
- **Companion-rail "Switch familiar" has no focus ring.** `companion-rail.tsx:77–85`.
- **Notification bell dropdown has no visible Escape hint and no close button.** `notification-bell.tsx:142–303`.
- **⌘K search button's visible label is redundant with `aria-label`.** `top-bar.tsx:52–61` — keep visible text, drop the aria-label.
- **Sidebar action row heights drift** (38px / 42px / 34px). `sidebar-minimal.css:578–590`.

### Agents / familiars

- **Familiar studio tabs are buttons, not `role="tab"`.** `familiar-studio.tsx:94–109` — no arrow-key cycling; disabled tabs are still in tab order.
- **Long familiar names overflow the detail header.** `agents-view.tsx:548–559` — missing `min-w-0` on the flex wrapper.
- **3D graph node selection isn't re-announced.** `memory-graph-3d.tsx:452` — aria-live fires only once at mount.
- **Memory graph rebuilds silently on filter change** (~500ms freeze on dense graphs). Add a "rebuilding…" status.
- **`SkillDetailDrawer` open/close is instant** while other drawers transition — pick one.
- **Eyebrow contrast pass missed `skill-card.tsx:114`** (still `--text-muted`, should be `--text-secondary`).

### Board / calendar / inbox

- **Board table has no keyboard nav.** `board-table.tsx` — arrow keys, Enter, Esc unimplemented.
- **Optimistic update reversal leaves no toast.** `board-view.tsx:94–105` — UI silently snaps back.
- **Kanban card focus ring disappears while dragging.** `board-kanban.tsx:163` (opacity 0.35 hides the ring).
- **Calendar month-view "+3 more" is muted and easily missed**, and lacks aria-label. `calendar-view.tsx:299–306`.
- **`new-card-modal` breadcrumb looks clickable but isn't.** `new-card-modal.tsx:111`.
- **Snooze menu can clip viewport on narrow widths.** `snooze-menu.tsx:51`.
- **Toast auto-dismiss is fixed 8s with no hover-pause** and no aria-live. `inbox-toast.tsx:24,42`.
- **Calendar arrow-key nav only at the macro level.** `calendar-view.tsx:810–826` — no vertical nav inside TimeGrid.

### Library / salem / browser / terminal

- **Reader has no j/k or arrow-key nav for headings.** `library-doc-preview.tsx` — slow in long docs.
- **Bookmarks / reading-list table rows are clickable but not keyboard-activatable.**
- **Browser address-bar suggestions lack `role="listbox"`/`aria-controls`/`aria-expanded`.** `browser-pane.tsx`.
- **Browser tabs missing `aria-current="page"` on the active tab.**
- **Browser quick-open rows are ~20px** — tight for pointer accuracy. `browser-quick-open.tsx:100+`.
- **GitHub error states aren't `role="alert"`.** `github-action-popover.tsx`, `library-github-list.tsx`.
- **Modal backdrops hardcoded** (`rgba(0,0,0,0.92)`, `bg-black/40`) — won't invert for light mode. `library.css`, `github-view.tsx`.
- **Salem perch is `div role="button"`** rather than a real `<button>`. `salem-widget.tsx:98`.

### Settings / aesthetic / onboarding

- **Settings nav has no `aria-label`.** `settings-shell.tsx:69`.
- **Onboarding form inputs aren't id-bound to their labels.** `onboarding-overlay.tsx:859–912`.
- **Theme preset swatches have `title` but no `aria-label`.** `settings-shell.tsx:488–527`.
- **Import URL error not `aria-describedby`-linked to the input.** `settings-shell.tsx:696–751`.
- **Glyph validation error isn't announced** (no aria-live). `onboarding-overlay.tsx:896`.
- **Vault required fields have no visual `*` and no `aria-required` indicator.** `vault-panel.tsx:82–129`.

- **Section grouping density drifts** between `--bg-card` and `--bg-raised`, py-3 vs py-2.

## P2 — polish

- **No `::selection` rule, no caret tokens.** Browser defaults vary across themes.
- **Disabled opacities differ across components** (0.25 / 0.3 / 0.4 / 0.5). Add `--opacity-disabled`.
- **No `--color-{success,warning,danger}-foreground` pairs** for light mode. Badge text-on-color inferred ad-hoc.
- **Focus ring offset is uniform `2px`** — icon buttons on raised surfaces could use `.focus-ring-inset` more consistently.
- **Reader measure can hit ~90ch** with the 820px modal + Lora 16/1.85 (reader spec moves to 820px). Add `max-width: 65ch` on `.library-preview-md`.
- **Filter-chip removal has no undo** (`board.css:55–60`). Cheap to add.
- **Inspector vs. modal pattern inconsistency** across board / chat / familiar studio — pick "slide-in panel" or "centred modal" per use-case and stay consistent.
- **Plugin cards and skill cards don't share a card token.** No `.surface-card` primitive.
- **Truncated timeline rows have no `title` attribute.** `library-timeline-row.tsx:90–92`.
- **Notification dropdown has no close button** — Esc works but isn't surfaced.
- **`scrollIntoView` calls don't announce** — fine for sighted, mute for SR.

## What's already strong

Worth naming, because the polish work should preserve it.

- **Token architecture in OKLCH is coherent.** Light/dark overrides derive from a single hue; semantic roles (success/warning/danger/info + soft variants) are present.
- **Motion language has three durations + three easings** and is honored in the shell.
- **`focus-ring` / `focus-ring-inset` classes are widely applied** — most chrome buttons, list rows, and inputs already get a visible 3px outline. The recent #253/254/255 sweep raised the floor materially.
- **Semantic landmarks exist** — `<nav>`, `<main>`, `<aside>`, `role="alert"`/`status` are used where they're used.
- **Keyboard shortcuts are documented** — ⌘K, ⌘B, ⌘1–9, ⌥1–9, j/k in escalations are all wired and discoverable.
- **Inbox escalations is the model for the rest of the app** — j/k nav + e/r/x/s/o actions + footer cheat-sheet is the keyboard pattern other surfaces should adopt.
- **3D scenes dispose cleanly** — no GPU leaks.
- **Theme-script prevents flash-of-wrong-theme** at boot.
- **Error retry UX in library is solid** (`library-error-retry.test.ts` documents the contract).
- **Existing tests document polish intent** — `chat-view-polish.test.ts`, `onboarding-polish.test.ts`, `agents-view-stats.test.ts`, `coven-floor-table.test.ts`. Honor these as living acceptance criteria.

## Suggested order of attack

If we wanted one PR per band:

1. **Foundations PR** — `useFocusTrap` hook, `<LiveRegion>` component, `useRovingTabIndex` hook, `<EmptyState>` / `<ErrorState>` primitives, `--scrollbar-thumb`/`--opacity-disabled`/`::selection` tokens. No surface changes, but every subsequent PR uses them.
2. **A11y P0 PR** — apply foundations to: command palette (listbox), board (focus trap + table keyboard + drag-alt menu), library reader (focus trap), chat (aria-live + textarea labels), glyph picker (roving tabindex), onboarding (focus trap + Escape), xterm SR mirror.
3. **Motion-language PR** — wrap every keyframe and named transition in the existing reduced-motion media query; add a single `@media (prefers-reduced-motion: reduce)` rule for `.ui-btn-spinner`, `.board-drawer-in`, `.command-palette-fade`, `ThinkingIndicator`, `salem-widget`, smooth-scroll calls.
4. **Token completion PR** — Salem section migration, kill remaining `text-white`/`bg-white`/`text-black`/`bg-black`, tokenize modal backdrops, derive light-mode `--ring-focus`.
5. **Per-surface polish** — chat (new-msg pill + role=alert errors + code-copy visible), board (optimistic-rollback toast, kanban focus-while-dragging, snooze positioning), library (j/k + 65ch measure), settings (form labelling + theme grid breakpoint + aesthetic mode picker).

This deliberately defers the [shell IA redesign](./2026-06-08-ui-ux-shell-ia-design.md) — that work absorbs several of the P1 shell issues, and doing the foundations first means the redesign inherits them rather than re-implementing them.
